The multilinear polynomial ring used throughout the paper also has fixed variable ordering (i.e. we can't have x1x2 and x2x1 in the polynomial for thm 6.1 to hold, and in that case it's not because x2x1 = x1x2, but really because x2x1 is not allowed). I disagree with "we can't have x1x2 and x2x1 in the polynomial for thm 6.1 to hold" because we can and want to: For example we can have the Hamiltonian H = x1 x2 + x2 x1, and we don't want to delete one of the two terms. For normal form purposes, the lexicographical ordering of x1 x2 is before x2 x1, just like how "ab" is alphabetically before "ba" but both are valid sequences of letters. The ring of controlled states is commutative because we always call the first output x1, the second output x2, and so on. But for the ring of controlled matrices is non-commutative: the output called x1 is connected to a particular controlled box, and the output called x2 is connected to a different controlled box, so when you change the order of the outputs you still need to bookkeep which output is connected to which box.
